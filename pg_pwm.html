PWM Layout

Pin 1 should be labeled such on the PCB and should also be the closest pin to the SD card. The is one PWM pin available on the Raspberry Pi which is GPIO18 or pin 12 on the Raspberry Pi's physicial header.

PWM Operation

There are two main modes of operation for the PWM module, PWM and Serial. PWM mode is further divided into two submodes, one which uses an algorithm to "even out" the pulses, and the other which has a more defined mark/space ratio.

PWM Operation: PWM

The PWM mode of operation is chosen by setting the mainMode argument of gpioPwmSetModes() to be pwmMode defined in ePwmMode. There are two available submodes to this, pwmAlgorithm and msTransmission.

  • PWM with pwmAlgorithm
    When PWM mode of operation is set to pwmAlgorithm the PWM module uses an algorithm to "spread" out the desired data as evenly as possible. For example if gpioPwmSetData() is called to set the data to 4 and gpioPwmSetRange() is called to set the range to 8 the output from the PWM pin for one period will be: 01010101
  • PWM with msTransmission
    When PWM mode of operation is set to msTransmission the data is grouped together and sent as a single mark-space per period. For example if gpioPwmSetData() is called to set the data to 4 and gpioPwmSetRange() is called to set the range to 8 the output from the PWM pin for one period will be: 11110000

PWM Operation: Serialiser

If mainMode argument of gpioPwmSetModes() is set as serialiserMode the data set by gpioPwmSetData() is sent serially. For example if gpioPwmSetData() is called with data argument set as 0x0F0F the output from the PWM pin for one period would be: 0000 0000 0000 0000 0000 1111 0000 1111
